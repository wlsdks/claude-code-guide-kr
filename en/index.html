What is Claude Code?

Even if you never write code yourself, just ask in plain words and it reads, edits, and runs your files for you. It's an AI tool.

Let's start by seeing how it differs from the chat you're used to (ChatGPT, the Claude chat app).

The point isn't just 'does it give you an answer' — it's does it actually build it in your project and make it work. Today's chat tools can edit text and code too, but that's 'shown inside the chat.' Claude Code creates several files itself, connects them, runs them, and finishes a 'working result.' (Both use the same Claude model.)

Tokens

A token is the smallest piece an AI reads text in (it can be smaller than a word). Like a taxi meter, the longer the text the higher it climbs — and your cost and usage limits are all counted in these.

An AI doesn't read characters as-is — it splits text into tokens. Try typing something.

    CLAUDE.md

    CLAUDE.md is 'a note you write yourself.' Claude reads it automatically at the start of every conversation.

    Write down your project rules and tone once, and you won't have to explain them every time.

    CLAUDE.md example

    # Reply rules - Always in English, in a polite tone - Explanation before code # This project - The About page is in the pages/about folder - Keep colors in the blue family
    • Keep it short, under 200 lines.If it's too long, it actually gets followed less.
    • You can put it in several places.One for your whole computer (~/.claude/CLAUDE.md) plus per-project (./CLAUDE.md) — they merge and load together.
    • It's shared with your team.A project's CLAUDE.md is the shared rules. For personal settings, use CLAUDE.local.md.

    Memory

    Automatic memory is 'a note Claude writes itself.' It records what it learns even without being told to.

    If CLAUDE.md is the note you write, this is the note Claude writes. Let's play through how it's saved and reused.

    1. You say something
    "From now on, always lay reports out as tables"
    ↓
    2. Claude records it in a note file (.md)
    Saved in ~/.claude/projects/…/memory/
    ↓
    3. The next day, a new session starts
    MEMORY.md is reloaded automatically (without you saying so)
    ↓
    4. It lays it out as a table on its own
    "I've made the report into a table" — remembered without being reminded

    Here's how it's actually recorded

    # MEMORY.md (written automatically by Claude) (Press the button to see it being recorded)

    Memory is just a plain Markdown file, so you can view and edit it directly with /memory. (Automatic memory requires Claude Code v2.1.59 or later.)

    "Can it reference older content across multiple sessions?" — the question non-developers ask most

    • Sessions are independent of each other.A new session doesn't carry over the previous conversation as-is (it starts clean).
    • Instead, memory and CLAUDE.md act as a 'bridge.'They're reloaded automatically at the start of each session, so preferences and rules learned earlier carry forward.
    • To continue one specific old conversation in fulluse /resume to pick it from a list; for your most recent one, use claude --continue.
    • It's saved only on your computer.It's not auto-shared to the cloud, other devices, or coworkers. To share rules with your team, write them in CLAUDE.md.

    In short: continue one conversation as-is = /resume, carry what was learned into the next conversation = memory and CLAUDE.md. (Source: official docs, as of June 2026.)

    Skills

    A skill is, in a word, 'your own recipe' — write down a task you do often, make it once, and reuse it again and again.

    Let's play through the difference between explaining from scratch each time and setting it once as a skill.

    Without a skill Repeat every time

    Using a skill Define once

    A skill opens only when it's needed, so when it's not in use it takes up almost no context window. If CLAUDE.md is an 'always-on note,' a skill is a 'manual you open when you need it.'

    What it looks like

    # .claude/skills/weekly-report/SKILL.md --- name: weekly-report description: Write the weekly report in the company format --- # Steps 1. Gather what was done this week 2. Fit it to the company format (header, tables) 3. Put the key summary at the top

    Invoke it: call it directly like /weekly-report, or for related tasks Claude applies it on its own. Pre-made skills can also be installed as plugins (the 'Plugins' tab).

    Subagents

    A subagent is, in a word, 'a worker you hand a job to' — give it a specific task and it works in its own workspace, reporting back only the result.

    The idea is simple — 'divide the work and hand it off.'

    An easy analogy · Picture yourself as a team lead. Instead of doing a big job alone, you hand pieces to specialist teammates: "You research this," "You review that." Each works at their own desk and reports back only the result. Your desk (the main conversation) stays clean.

    Automation (hooks)

    A hook is, in a word, 'an automatic rule' — at a specific moment, a set action runs automatically, every time.

    Like an 'auto-reply email,' it kicks in on its own when something happens.

    RelaxYou don't need to build it yourself. Setting it up is a little technical, but non-developers don't have to know how — just ask Claude in plain words, like "Make a hook that auto-tidies every time I edit a file," and Claude builds it for you. For this page, knowing 'this exists' is enough.

    How it works When → what → how

    1
    Event (when)
    e.g. right after a file is saved
    2
    Filter (what)
    e.g. if it's an important document
    3
    Action (how)
    e.g. automatically make a backup copy

    Great for things like

    Auto-backup
    Automatically back up an important file every time it changes.
    Protection
    Prevent accidentally deleting an important file.
    Notifications
    Send an alert when a job finishes.
    • You write it in a settings file (settings.json).You can set it at several levels: all projects, this project, or personal.
    • It's guaranteed every time.If CLAUDE.md is a 'request,' a hook is a 'rule' — it never forgets and always runs.
    • Supports dozens of lifecycle points.file changes, before/after tool runs, session start, before/after compaction, and more · check with /hooks.

    Since the setup is a bit technical, just ask Claude something like "Make a hook that formats every time I edit a file" and it builds it for you. (Source: official docs, as of June 2026.)

    Plan mode

    Before making any changes, Claude shows you 'what it plans to do' first and gets your approval.

    For big changes or unfamiliar work, this heads off going 'in the wrong direction.' Play through the flow.

    1
    Switch to plan mode
    Shift+Tab or /plan — now it only reads and doesn't edit
    2
    Claude presents a plan
    "I'll change these 3 files like this" — shows the impact and order up front
    3
    You review and adjust
    Refine it with feedback like "You missed the footer too"
    4
    Approve, then it runs
    "Okay, go" → then it edits the files all at once

    Plan mode is 'read-only,' so it's safe. It's one of the permission modes (the 'Permission modes' tab). (On some Windows terminals, /plan is recommended instead of Shift+Tab.)

    Permission modes

    You decide how much Claude does on its own.

    In plain terms · It's a setting for how often Claude asks "May I do this?" when it does something on your computer (editing files, running commands). At first it's safer to ask every time; once you're comfortable, you can let it handle frequent tasks on its own. It's like a car's 'manual ↔ automatic.'

    In Claude Code, Shift+Tab cycles through the default modes (ask permission, accept edits, plan). Autonomous modes like 'auto / skip' may need a separate option. Pick a mode and use "See it work" to check the difference.

    ← Safe · asks a lotAutonomous · asks little →

    These 5 are the modes shown in the menu. There's also a separate automation-only mode that 'only allows pre-approved tools (dontAsk).' In any mode, extreme destruction like wiping the whole system (rm -rf / · rm -rf ~) still asks for one more confirmation (a 'circuit breaker' — they don't run automatically). If you don't like a result, you can recover it with 'rewind' (the 'Rewind' tab).

    Rewind (checkpoints)

    Claude auto-saves the state at the moment of every request you send. If you don't like a result, you can roll back.

    It's like 'time travel' — you can go to an earlier point. Click a dot to roll back.

    Scenario: "Make the design more modern" → 10 files changed, but you don't like it. How do you roll back?

    Click a dot on the timeline above, then choose how to restore.

    • How to open it/rewind, or press ESC twice when the input box is empty — the list of your requests appears. (If there's text in the input, double-ESC clears the text first.)
    • Three kinds of restoreCode + conversation together / conversation only (keep code) / code only (keep conversation).
    • LimitsThings changed by terminal commands like rm·mv, installs (npm install), and outside changes can't be rolled back.

    Checkpoints are 'quick rollback within a session' (kept about 30 days); git is 'permanent version control' — they serve different roles. To keep something long-term, commit it to git. (Source: official docs, as of June 2026.)

    Subscription vs API — what's different

    Subscription (Pro, Max, Team)

    Monthly fee
    CostA set amount each month
    LimitA fixed amount of usage — once used up, wait for the reset
    Good whenYou use it steadily and often (a flat fee is reassuring)

    API (pay-as-you-go)

    Billed for what you use
    CostCharged only for the tokens you use
    LimitNo monthly cap — you can keep using it, but the cost rises
    Good whenYou use it occasionally, or for automation and integrations

    There are two limits at the same time

    • 5-hour limitLooks at how much you've used in 'the last 5 hours.' It refills automatically over time (rolling).
    • Weekly limitA larger weekly cap that resets at a fixed day/time each week (it doesn't roll like the 5-hour one). Avoiding the 5-hour cap alone won't let you use it forever. (Max has two: an all-models one + a Sonnet-only one.)
    • If either one fills up, it stops.Both the 5-hour and weekly limits apply, so whichever fills first blocks you. When that limit resets, it frees up again.
    • Hitting a limit doesn't lose your work.What you were doing isn't wiped — it's just a 'brief pause.' When it resets, pick up where you left off. (In a hurry? Upgrade to a higher plan, or buy extra usage.)
    • It's deducted from one shared place.claude.ai, Claude Code, and the desktop app share the same usage.

    How to conserve your limit

    Use a cheaper model
    Use Haiku for simple jobs. Opus is heavy and pricey. (the 'Choosing a model' tab)
    Clear often
    Tidy the context with /clear and /compact. (the 'Context window' · 'Sessions' tabs)
    Ask concisely
    Request only the files and essentials you need.

    Exact times, prices, and limit figures vary by plan and time — check at claude.com/pricing and support.claude.com. (The concepts are based on the official docs as of June 2026.)

    Why manage context?

    Context is Claude's "desk." When the desk gets cluttered, work gets slower, pricier, and more error-prone. That's why tidying up at the right time matters.

    See for yourself: how things get worse if you "keep stacking the conversation," and how they get better when you press "tidy up."

    How full the context is12%
    Slowdown — it re-reads everything each timeLow
    Token cost — the more there is, the pricierLow
    Confusion — important things get missed among the clutterLow

    Right now the desk is clean — fast, cheap, and accurate.

    How to manage — tidy up like this

    New task = /clear
    Clear the desk when moving to a different task. (See the "Sessions & folders" tab.)
    Getting long? /compact
    Summarize just the essentials to free up space. (See the "Compacting" tab.)
    One task per session
    Don't mix several jobs — mixing makes things confusing.
    Load only what's needed
    Don't add files or content you won't use.
    • Why does it get slower and pricier?Claude re-reads the whole desk every time it answers. The more it piles up, the longer each turn takes and the more tokens (= cost) it uses.
    • Why does accuracy drop?When the context is too long, important details in the middle are easy to lose. Keeping only what's needed makes it more accurate.
    • Neglect it and auto-compact kicks in.When it fills up, Claude summarizes on its own (auto-compact), and some details can disappear then. Tidy up ahead of time and you stay in control. (See the "Context window" and "Compacting" tabs.)

    In short — a clean desk = fast, accurate work. Just remember /clear when the task changes and /compact when it gets long.

    Thinking depth (effort)

    Thinking depth (effort) is the setting that decides how deeply Claude thinks before answering. The deeper it thinks, the more accurate it is on hard problems — but it's also slower and more expensive.

    There are low · medium · high · x-high · max levels (the default is 'high'). The deeper it thinks, the more 'thinking tokens' it uses (see the 'Tokens' tab). Pick one and see the difference.

    Accuracy on hard problems
    Response speed
    Token cost

    The deeper you go, the longer the 'thinking process' try it — the length changes

    Use 'low' for simple tasks (fast and cheap), and 'high/max' for tricky analysis and reasoning. How to change it: /effort or settings (the 'Settings' tab). example The bars are a relative comparison. (Source: Anthropic official — support.claude.com model & effort settings)

    Advanced · ultracode — The /effort menu also has ultracode. On top of maximum (xhigh) reasoning, it has Claude automatically marshal multiple helpers (a workflow) for hard tasks. It's session-only, uses a lot of tokens, so you don't need it day-to-day — pick it only for very hard tasks (on Fable 5, Opus 4.8, etc.).

    Goals (goal)

    Goals (goal) is a feature where you give only a 'completion condition' and Claude keeps repeating on its own until that condition is met. No need to ask again every time.

    Instead of doing it once and stopping, it judges for itself after every turn whether the condition is satisfied, and if not, it does it again. See below.

    Example goal"Fix this notice until there are 0 typos"
    Repeats automatically until the condition is met

    Press "Start goal" to try it.

    It checks the condition with a separate judgment every turn, and stops automatically once it's met (there's also a safeguard against infinite repetition). How to change it: /goal. (Source: Anthropic official — code.claude.com/docs goal)

    Repeat (loop)

    Repeat (loop) is a feature that automatically re-runs the same task at a set interval. Example: every 10 minutes, "check for new orders and notify."

    If goals run 'until a condition is met,' repeat runs 'at a fixed time interval.' Pick an interval and start. (The simulation fast-forwards time to show it.)

    Run history

    Pick an interval and press "Start repeat".

    It repeats while the session stays open. How to change it: something like /loop 10m "check new orders". (Source: Anthropic official — code.claude.com/docs scheduled-tasks)

    Workflows Advanced

    Workflows is a feature where Claude plans its own 'strategy' for a really big job and runs several helpers (subagents) at once to handle it all together.

    If subagents are 'handing it to one person,' workflows are 'coordinating several people at once.' Great for big reviews, cleanups, and research (higher plans like Max & Team).

    Example"Review 8 contracts item by item, all at once" → Claude plans a strategy and deploys several helpers in one go

    Press "See parallel work" to try it.

    For non-developers this is a somewhat advanced feature — just knowing 'this is possible too' is plenty. Since it uses many helpers at once, it uses a lot of tokens, so it's only suited to big jobs. (Source: Anthropic official — claude.com dynamic workflows intro)

    Settings (config)

    Settings (config) is a control panel where you change things like language, theme, model, thinking depth, and permissions all in one place.

    Things like "I want to change the language" all happen here. Press a value to change it (an example panel to help you understand).

    Press a value on the right to change it.

    How to open it: /config. There are actually more settings than this. (Source: Anthropic official — code.claude.com/docs settings)

    Diagnostics (doctor)

    Diagnostics (doctor) is a tool that checks your install, login, connections, and settings all at once when something isn't working. The 'first thing to try when something's broken.'

    Like a car's check-engine light, it points out where the problem is. Try running diagnostics yourself.

    Running /doctor checks

    Press "Run diagnostics" to try it.

    How to open it: /doctor (or claude doctor). If there's a problem, it also tells you how to fix it. (Source: Anthropic official — code.claude.com/docs CLI reference)

    Glossary

    A one-line summary of the terms that came up in this guide.

    When you're confused, look it up quickly here. Type a word in the search box and it filters right away.

    If unsureFour that look similar — Skill=your own recipe · Plugin=a box of features · Subagent=a worker you hand a job to · Hook=an automatic rule. Similar names, different roles.
    Context window
    The 'workspace' Claude holds at one time. It only knows what's in here. (the 'Context window' tab)
    Token
    The smallest piece an AI reads text in. The unit for context and cost. (the 'Tokens' tab)
    Session
    One continuous conversation. Everything piles up in its context window. (the 'Sessions & folders' tab)
    CLAUDE.md
    A rules note you write. Loaded automatically in every conversation. (the 'CLAUDE.md' tab)
    Memory
    A learning note Claude writes itself. Carries over across sessions. (the 'Memory' tab)
    Skill
    A bundle of steps for a frequent task. Unfolds only when needed. (the 'Skills' tab)
    MCP
    The standard for connecting to outside tools (Jira, Figma, etc.). (the 'MCP' tab)
    Plugin
    A bundle that packages skills, MCP, commands, and more for distribution. (the 'Plugins' tab)
    Subagent
    A specialist helper that takes on a specific job. (the 'Subagents' tab)
    Hooks
    Rules that run automatically at a specific moment. (the 'Automation (hooks)' tab)
    Plan mode
    A mode that shows a plan and gets approval before editing. (the 'Plan mode' tab)
    Permission modes
    A setting for how often it asks. (the 'Permission modes' tab)
    Checkpoint · Rewind
    Auto-saves at every request; restore to an earlier point. (the 'Rewind' tab)
    Compaction (compact)
    Summarizes the conversation to free up context room. (the 'Context window' tab)
    Models (Opus, Sonnet, Haiku)
    Claudes that differ in smartness, speed, and cost. (the 'Choosing a model' tab)
    Terminal (CLI)
    The black input window where you instruct a computer with text (commands). Also called the 'command line.' CLI = Command Line Interface.
    API
    A 'connection pathway' that programs use to talk to each other. Here it means the 'pay-as-you-go' way of using it. (the 'Plans & pricing' tab)
    Server
    A computer that does work for you from afar. An MCP 'server' is a small program that connects an outside tool to Claude.
    .md (Markdown)
    A 'note-style text file' format styled with simple rules. The .md in CLAUDE.md and SKILL.md is this.
    JSON
    A computer-readable 'format for writing settings.' Settings like hooks go here. (You don't have to write it yourself — Claude does it for you.)
    git
    A 'version control' tool that records and reverts file changes. The permanent version of rewind. (the 'Rewind' tab)
    Pay-as-you-go
    Paying 'for what you use' instead of a monthly fee (API). (the 'Plans & pricing' · 'Usage limits' tabs)
    Rolling limit
    A usage limit that flows along and refills, like 'the last 5 hours' or 'the last 7 days.' (the 'Usage limits' tab)
    Protocol
    A 'common standard' that lets different programs talk to each other. This is the P in MCP.
