클로드 코드란?

코드를 직접 쓰지 않아도, 평소 말로 부탁하면 파일을 읽고 고치고 실행해 주는 AI 도구예요.

평소 쓰던 챗(ChatGPT·클로드 챗)과 뭐가 다른지부터 눈으로 비교해 볼게요.

챗 vs 클로드 코드 같은 부탁 — 눌러보세요

클로드 챗 조언·코드 조각

우리 홈페이지에 예약 기능 추가해줘

클로드 코드 작동하는 결과물

우리 홈페이지에 예약 기능 추가해줘

핵심은 ‘답을 해주느냐’가 아니라 내 프로젝트에서 실제로 만들어 작동까지 시키느냐예요. 요즘 챗도 글·코드를 고쳐주지만 그건 ‘채팅 안에서 보여주는 것’ — 여러 파일을 직접 만들고, 연결하고, 실행해서 ‘작동하는 결과물’을 완성하는 게 클로드 코드예요. (둘 다 같은 클로드 모델을 써요.)

처음이라면 ‘터미널’이나 ‘VS Code 확장’으로 시작하는 경우가 많아요.

자동 압축(auto-compact)이란?

대화가 길어져 컨텍스트 창이 한계에 가까워지면, 클로드가 스스로 지난 대화를 요약해 공간을 만들어요. 위 “자동 압축 보기”로 차오르다 요약으로 줄어드는 과정을 볼 수 있어요.

언제 시작되나? 컨텍스트 창이 거의 꽉 찼을 때(한계 근처) 자동으로 시작돼요. 정확한 기준 %는 공식적으로 공개돼 있지 않아요(“한계에 가까워지면”이라고만 안내). 압축 뒤에도 시스템 지침·CLAUDE.md·메모리·MCP 도구는 자동으로 다시 불러오지만, 처음 보였던 스킬 목록은 다시 채워지지 않고 ‘이미 사용한 스킬’만 남아요. 수동 /compact 도 같은 원리예요. (출처: 공식 문서, 2026년 6월 기준)

세션 & 폴더

세션은 한 번의 연속된 대화예요. 그 세션의 컨텍스트 창에 모든 게 쌓여요.

컴퓨터를 잘 몰라도 괜찮아요. 순서대로 — 먼저 폴더를 정하고, 그 안에서 세션을 시작해요.

1단계 — 작업 폴더부터 정하기 세션은 폴더 안에서 시작돼요

클로드는 시작한 폴더 안의 파일만 봐요. 그래서 ‘어느 폴더에서 시작하느냐’가 먼저예요. 안 정하면 어떻게 되는지 버튼으로 보세요.

버튼을 누르면 클로드가 ‘어떤 파일을 보게 되는지’ 나타나요.

2단계 — 그 폴더에서 세션 시작 한 세션에 여러 일을 섞으면?

아직 비어 있어요

위 버튼을 눌러 시작해 보세요.

방법하는 일언제

    • 200줄 이하로 짧게.너무 길면 오히려 잘 안 지켜요.
    • 여러 곳에 둘 수 있어요.내 컴퓨터 전체(~/.claude/CLAUDE.md) + 프로젝트별(./CLAUDE.md)이 합쳐져 함께 로드돼요.
    • 팀과 공유돼요.프로젝트의 CLAUDE.md는 같이 쓰는 규칙. 개인 설정은 CLAUDE.local.md.

    만들기: /init (프로젝트 분석해 자동 생성) · 편집: /memory. ‘클로드가 스스로 적는 메모’는 다음 장(08 메모리)에서.

    메모리

    자동 메모리는 ‘클로드가 스스로 적는 메모’예요. 내가 시키지 않아도 배운 걸 기록해 둬요.

    CLAUDE.md가 내가 쓰는 메모라면, 이건 클로드가 쓰는 메모예요. 어떻게 저장되고 다시 쓰이는지 재생해 볼게요.

    1. 내가 한마디 한다
    “앞으로 보고서는 항상 표로 정리해줘”
    ↓
    2. 클로드가 메모 파일(.md)에 기록
    ~/.claude/projects/…/memory/ 에 저장
    ↓
    3. 다음 날, 새 세션 시작
    MEMORY.md를 자동으로 다시 불러옴 (내가 말 안 해도)
    ↓
    4. 알아서 표로 정리해 준다
    “보고서를 표로 만들어 드렸어요” — 다시 안 알려줘도 기억

    실제로 이렇게 기록돼요

    # MEMORY.md (클로드가 자동 작성) (버튼을 누르면 기록되는 모습이 나타나요)

    메모리는 평범한 마크다운 파일이라 /memory 로 직접 보고 고칠 수 있어요. (자동 메모리는 Claude Code v2.1.59 이상)

    “여러 세션에서 예전 내용을 참조할 수 있나요?” — 비개발자분들이 가장 많이 묻는 질문

    • 세션은 서로 독립적이에요.새 세션은 이전 대화 기록을 그대로 가져오지 않아요(깨끗하게 시작).
    • 대신 메모리·CLAUDE.md가 ‘다리’ 역할을 해요.매 세션 시작 때 자동으로 다시 불러와서, 예전에 배운 선호·규칙은 이어져요.
    • 특정 옛 대화를 통째로 이어가려면/resume (또는 claude --continue)로 그 세션을 그대로 되살릴 수 있어요.
    • 내 컴퓨터에만 저장돼요.클라우드나 다른 기기·동료와 자동 공유되지 않아요. 팀과 나눌 규칙은 CLAUDE.md에 적어 공유해요.

    정리: 한 대화를 그대로 이어가기 = /resume, 배운 점을 다음 대화로 넘기기 = 메모리·CLAUDE.md. (출처: 공식 문서, 2026년 6월 기준)

    스킬 (Skills)

    스킬은 자주 쓰는 작업 방법을 적어둔 ‘요령 묶음’이에요. 한 번 만들면 계속 재사용해요.

    매번 처음부터 설명하는 것과, 스킬로 한 번 정해두는 것의 차이를 재생해 볼게요.

    스킬 없이 매번 반복

    스킬 사용 한 번 정의

    스킬은 필요할 때만 펼쳐지는(점진적 공개) 방식이라, 안 쓸 땐 컨텍스트 창을 거의 차지하지 않아요. CLAUDE.md가 ‘항상 켜진 메모’라면 스킬은 ‘필요할 때 펴보는 매뉴얼’.

    어떻게 생겼나

    # .claude/skills/주간보고/SKILL.md --- name: 주간보고 description: 주간 보고서를 회사 양식으로 작성 --- # 절차 1. 이번 주 한 일을 모은다 2. 회사 양식(머리말·표)에 맞춘다 3. 핵심 요약을 맨 위에 둔다

    호출: /주간보고 처럼 직접 부르거나, 관련 작업이면 클로드가 알아서 적용. 미리 만들어진 스킬은 플러그인으로 설치할 수도 있어요(11번).

    서브에이전트

    서브에이전트는 특정 일을 따로 맡는 ‘전문 도우미’예요. 자기만의 작업 공간에서 일하고 결과만 보고해요.

    핵심은 단순해요 — ‘일을 나눠 맡기기’.

    쉬운 비유 · 내가 팀장이라고 생각해 보세요. 큰 일을 혼자 다 하지 않고 전문 팀원에게 “이건 네가 조사해와”, “이건 네가 검토해줘” 하고 나눠 맡기죠. 팀원은 각자 자기 책상에서 일하고 결과만 보고해요. 내 책상(메인 대화)은 깨끗하게 유지되고요.

    직접 보기 — 메인 vs 서브에이전트 눌러서 재생

    메인 대화 (나와 클로드)컨텍스트 8%
    “이 프로젝트 조사하고 보안 검토해줘”
    조사 담당 (서브에이전트)
    자기만의 새 컨텍스트 창
    0토큰 · 파일 30개 읽는 중
    → 요약 600토큰만 메인에 보고
    보안 검토 (서브에이전트)
    자기만의 새 컨텍스트 창
    0토큰 · 코드 점검 중
    → 요약 500토큰만 메인에 보고

    “동작 보기”를 눌러보세요.

    • 자기만의 작업 공간(컨텍스트 창)을 가져요.도우미가 길게 뒤져도 본 대화는 안 지저분해져요.
    • 할 수 있는 일을 제한할 수 있어요.예: ‘코드 검토’ 도우미는 읽기만 하고 못 고치게 — 더 안전.
    • 기본 도우미가 둘 있어요: Explore(탐색)·Plan(계획).그 외엔 직접 만들 수 있어요(.claude/agents/).

    여러 세션을 동시에 돌리며 협업하는 ‘에이전트 팀(agent teams)’은 별도의 실험 기능이에요. (출처: 공식 문서)

    클로드에게 이미지 붙여넣기

    클로드는 이미지를 볼 수 있어요. 스크린샷을 붙이면 말로 설명하는 것보다 훨씬 빠르고 정확해요.

    버튼을 누르면 클로드가 이미지를 어떻게 ‘읽어내는지’ 보여드려요.

    로그인
    이메일
    비밀번호
    로그인
    비밀번호가 틀렸습니다

    클로드가 인식한 것

    • ✓글자 읽기 — 제목 “로그인”
    • ✓구조 파악 — 이메일·비밀번호 입력칸 2개
    • ✓색상 감지 — 파란색 버튼 “로그인”
    • ✓상태 인식 — 빨간 오류 “비밀번호가 틀렸습니다”
    → 클로드: “로그인 화면이고, 비밀번호 오류 상태군요. 검증 로직을 확인해 고쳐볼게요.”

    왜 캡처를 붙일까요?

    말보다 빠름
    화면을 길게 설명할 필요 없이 그대로.
    정확한 상태
    실제로 보이는 그 모습을 직접 확인.
    디자인 그대로
    시안을 주면 그 레이아웃·색으로.
    에러 화면
    오류 캡처를 주면 원인을 빨리.
    • 이미지를 ‘클릭’하진 못해요.화면을 분석할 뿐, 그림 속 버튼을 직접 누르진 않아요.
    • 아주 작은 글씨는 틀릴 수 있어요.중요한 숫자·글자는 텍스트로도 함께 적어주면 안전해요.
    • 움직이는 GIF는 첫 장면만 봐요.JPG·PNG 등 일반 이미지를 권장.

    붙이는 법: 캡처를 입력창에 끌어다 놓거나 붙여넣기(Cmd/Ctrl+V), 또는 이미지 파일 경로를 적어주면 돼요.

    자동화 (훅 · Hooks)

    훅은 특정 순간에 정해진 동작이 매번 자동으로 실행되게 하는 기능이에요.

    ‘자동응답 메일’처럼, 어떤 일이 생기면 알아서 작동해요. 조금 기술적이라 ‘이런 게 있다’만 알아도 좋아요.

    작동 방식

    1
    이벤트 (언제)
    예: 파일을 저장한 직후
    2
    필터 (무엇을)
    예: 그게 코드 파일이면
    3
    작업 (어떻게)
    예: 자동으로 코드 정렬·검사 실행

    이런 데 좋아요

    자동 정리
    고칠 때마다 자동으로 코드 정렬.
    보호
    위험한 명령(rm 등)을 미리 막기.
    알림
    작업이 끝나면 알림 보내기.
    • 설정 파일(settings.json)에 적어요.전체 프로젝트용·이 프로젝트용·개인용 등 여러 단계로 둘 수 있어요.
    • 매번 보장돼요.CLAUDE.md가 ‘부탁’이라면 훅은 ‘규칙’ — 잊지 않고 항상 실행돼요.
    • 26가지 넘는 시점을 지원해요.파일 변경·도구 실행 전후·세션 시작 등. /hooks 로 확인.

    설정이 다소 기술적이라, 클로드에게 “파일 고칠 때마다 정렬하는 훅 만들어줘”처럼 부탁하면 대신 만들어줘요. (출처: 공식 문서, 2026년 6월 기준)

    계획 모드

    바로 고치기 전에, 클로드가 ‘무엇을 할지’ 계획부터 보여주고 내 승인을 받아요.

    큰 변경이나 낯선 작업에서 ‘엉뚱한 방향’을 미리 막을 수 있어요. 흐름을 재생해 보세요.

    1
    계획 모드로 전환
    Shift+Tab 또는 /plan — 이제 읽기만 하고 고치지 않아요
    2
    클로드가 계획 제시
    “이 3개 파일을 이렇게 바꾸겠습니다” — 영향과 순서를 미리 보여줌
    3
    내가 검토 · 수정
    “footer도 빼먹었네” 같은 피드백으로 다시 다듬기
    4
    승인하면 실행
    “좋아, 실행해” → 그때 한꺼번에 파일을 고쳐요

    계획 모드는 ‘읽기 전용’이라 안전해요. 권한 모드(16번)의 한 종류예요. (Windows 일부 터미널에선 Shift+Tab 대신 /plan 권장)

    권한 모드

    클로드가 어디까지 알아서 할지 내가 정해요.

    쉽게 말하면 · 클로드가 내 컴퓨터에서 뭔가(파일 고치기·명령 실행)를 할 때, “이거 해도 될까요?”라고 얼마나 자주 물어볼지 정하는 설정이에요. 처음엔 매번 물어보는 게 안전하고, 익숙해지면 자주 하는 일은 알아서 하게 풀어줄 수 있어요. 자동차의 ‘수동 ↔ 자동’ 같은 거예요.

    클로드 코드에서 Shift+Tab 으로 모드를 바꿔요. 모드를 누르고 “동작 보기”로 차이를 확인하세요.

    ← 안전 · 확인 많음자율 · 확인 적음 →

    이 5가지는 메뉴에 보이는 모드예요. 자동화 전용으로 ‘사전 승인한 도구만 허용(dontAsk)’ 모드도 따로 있어요. 어떤 모드든 시스템 전체 삭제(rm -rf /) 같은 극단적 파괴는 안전장치로 막혀요. 마음에 안 들면 ‘되돌리기’(17번)로 복구할 수 있어요.

    되돌리기 (체크포인트)

    클로드는 내가 보낸 요청마다 그 순간의 상태를 자동 저장해 둬요. 마음에 안 들면 되돌릴 수 있어요.

    ‘시간 여행’처럼 이전 시점으로 갈 수 있어요. 점을 눌러 되돌려 보세요.

    상황: “디자인을 모던하게 바꿔줘” → 파일 10개가 바뀌었는데 마음에 안 들어요. 어떻게 되돌릴까요?

    위 타임라인의 점을 누른 뒤, 복구 방식을 골라보세요.

    • 여는 법ESC 를 두 번 누르거나 /rewind — 지금까지 보낸 요청 목록이 떠요.
    • 세 가지 복구코드+대화 함께 / 대화만(코드 유지) / 코드만(대화 유지).
    • 한계rm·mv 같은 터미널 명령으로 바뀐 것, 설치(npm install)·외부 변경은 되돌릴 수 없어요.

    체크포인트는 ‘세션 안 빠른 되돌리기’(약 30일 보관), git은 ‘영구 버전 관리’ — 역할이 달라요. 오래 보관하려면 git에 커밋하세요. (출처: 공식 문서, 2026년 6월 기준)

    요금제

    클로드 코드는 유료 플랜이나 API로 쓸 수 있어요. 무료 플랜에는 포함되지 않아요.

    정확한 가격·사용량은 자주 바뀌니, 숫자는 공식 페이지에서 확인하세요.

    사용 한도

    구독은 월정액에 사용 한도가 있어요. 다 쓰면 리셋될 때까지 기다려야 해서, 관리가 중요해요.

    압축 전 — 대화 기록 (길게 쌓임)

    나“가입하면 환영 메일이 안 가요. 원인 찾아서 고쳐줘.”
    파일auth.js · mailer.js · config.js 를 읽음 (합쳐서 수백 줄)
    도구 결과테스트 실행 로그 120줄 — 실패: SMTP 연결 타임아웃 … (긴 오류 로그가 주르륵 …)
    클로드“메일 서버 주소 설정이 빠져 있었네요. config.js 에 SMTP 호스트를 추가하고, 누락된 발신자 주소도 채웠습니다. 이러이러한 이유로 … (긴 설명이 이어짐)”
    도구 결과테스트 재실행 로그 80줄 — 통과 ✓ (상세 로그 …)
    클로드“수정 완료. 이제 환영 메일이 정상 발송됩니다.”
    압축 전 약 12,000토큰→압축 후 약 1,400토큰 (약 88% 절약, 예시)
    요약 한 칸으로
    사용자가 ‘가입 시 환영 메일 미발송’ 해결을 요청. auth.js·mailer.js·config.js 확인 → config.js의 메일 서버(SMTP) 설정 누락이 원인. SMTP 호스트·발신자 주소를 추가하고 테스트 통과. 환영 메일 정상 발송 확인.
