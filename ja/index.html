結論チャットは方法・コード片を見せ — Claude Code は動く成果物まで作ります。

Cowork とは？

Cowork は、コードではなく 「オフィス業務」を代わりにこなしてくれる Claude です。目標を伝えるだけで、自分のファイルを直接扱って完成した成果物（Excel・文書など）を作ってくれます。

    • チームと共有されます。プロジェクトの CLAUDE.md は一緒に使うルール。個人設定は CLAUDE.local.md。

    作成: /init (プロジェクトを分析して自動生成) · 編集: /memory。「Claude が自分で書くメモ」は次の章 (08 メモリ) で。

    メモリ

    自動メモリは「Claude が自分で書くメモ」です。指示しなくても、学んだことを記録しておきます。

    CLAUDE.md が自分の書くメモなら、これは Claude が書くメモです。どう保存され、また使われるのか再生してみましょう。

    1. 自分がひとこと言う
    「これからレポートはいつも表で整理して」
    ↓
    2. Claude がメモファイル(.md)に記録
    ~/.claude/projects/…/memory/ に保存
    ↓
    3. 翌日、新しいセッションを開始
    MEMORY.md を自動で読み込み直す (自分が言わなくても)
    ↓
    4. 自動で表に整理してくれる
    「レポートを表にしました」 — もう一度伝えなくても覚えている

    実際にはこう記録されます

    # MEMORY.md (Claude が自動で作成) (ボタンを押すと、記録されていく様子が表示されます)

    メモリはふつうのマークダウンファイルなので、/memory で直接見て直せます。（自動メモリは Claude Code v2.1.59 以上）

    「複数のセッションで以前の内容を参照できますか?」 — 非エンジニアの方がいちばん多く聞く質問

    • セッションは互いに独立しています。新しいセッションは以前の会話の記録をそのまま持ってきません（きれいに開始）。
    • 代わりにメモリ・CLAUDE.md が「橋」の役割をします。セッションを始めるたびに自動で読み込み直すので、以前学んだ好みやルールは引き継がれます。
    • 特定の古い会話をまるごと続けたいなら/resume で一覧から選んで復活、直前の会話なら claude --continue で続けられます。
    • 自分のコンピュータにだけ保存されます。クラウドや他の端末・同僚と自動共有はされません。チームで分かち合うルールは CLAUDE.md に書いて共有します。

    まとめ: 一つの会話をそのまま続ける = /resume、学んだ点を次の会話へ引き継ぐ = メモリ・CLAUDE.md。（出典: 公式ドキュメント、2026年6月時点）

    スキル (Skills)

    スキルはひとことで言うと「自分のレシピ」です — よく使う作業の手順を書いておき、一度作ればずっと再利用できます。

    毎回ゼロから説明するのと、スキルで一度決めておくのとの違いを再生してみましょう。

    スキルなし 毎回くり返し

    スキルあり 一度だけ定義

    スキルは必要なときだけ開く方式なので、使わないときはコンテキストウィンドウをほとんど占めません。CLAUDE.md が「いつもオンのメモ」なら、スキルは「必要なときに開くマニュアル」です。

    どんな見た目?

    # .claude/skills/週報/SKILL.md --- name: 週報 description: 週次レポートを会社の様式で作成 --- # 手順 1. 今週やったことを集める 2. 会社の様式（見出し・表）に合わせる 3. 要点のまとめを一番上に置く

    呼び出し: /週報 のように直接呼ぶか、関連する作業なら Claude が自動で適用。あらかじめ作られたスキルはプラグインとしてインストールすることもできます（「プラグイン」タブ）。

    MCP (ツール連携)

    MCP は Claude が他のアプリやサービス（Jira・Google ドライブなど）を直接使えるようにつなぐものです — アプリ同士でアカウントを連携するようなイメージです。

    MCP = Model Context Protocol (モデルが外部とつながる標準)。ボタンを押すと連携の過程を見られます。

    かんたんにあるアプリに「Google でログイン」をつなぐと、そのアプリが自分の Google を使えますよね? MCP もそんな「連携」です — Claude に Jira・Figma のようなツールをつないであげると、Claude がそれを直接開いて仕事します。自分でいちいちコピペしなくても済むんです。
    左でツールを連携すると、Claude が何をもっとできるようになるかをここでお見せします。

    「ひとつずつ連携してみる」を押してみてください。

    こんなものを連携します (例)

    Google ドライブ・スプレッドシート
    書類や表を直接読んで整理します。
    例 「このシートの合計を出して」
    Slack
    メッセージを読んで送ります。
    例 「このお知らせを Slack に投稿して」
    Jira
    共同作業の課題を読んで作ります。
    例 「この件を Jira チケットに」
    その他
    Figma・ブラウザ・Notion・GitHub など。
    例 社内ツールと連携

    連携したツールは /mcp で確認。ツールが多いほど、その説明もコンテキストを少しずつ占めるので、必要なものだけ連携してください。MCP を複数まとめて配布するのが「プラグイン」です（「プラグイン」タブ）。

    プラグイン

    プラグインはひとことで言うと「機能のセット箱」です — いろいろな機能を一度にインストールする束（スマホの「アプリ」のように）。

    スキル・ツール連携(MCP)・自動化(フック)などをひとつひとつ別々にそろえる代わりに、誰かがうまく作っておいた束をまるごと受け取って使うわけです。

    かんたんにスマホに「写真補正アプリ」を一つ入れると、フィルターやツールが一気についてきますよね? プラグインもそうです。 これまで見たスキル・MCP・サブエージェント・フックを一つの束としてインストールするんです。（それぞれが何か全部わからなくても大丈夫 — ここでは「束」だということだけわかれば十分です。）

    サブエージェント

    サブエージェントはひとことで言うと「仕事を任せる社員」です — 特定の仕事を任せると自分専用の作業空間で処理し、結果だけを報告します。

    核心はシンプルです — 「仕事を分けて任せる」。

    わかりやすいたとえ · 自分がチームリーダーだと思ってください。大きな仕事を一人で全部やらず、専門のメンバーに「これは君が調べてきて」「これは君が確認して」と分けて任せますよね。メンバーはそれぞれ自分の机で働き、結果だけ報告します。自分の机（メインの会話）はきれいに保たれます。

    実際に見る — 6 人に分けて任せる 押して再生

    メインの会話 (自分と Claude)コンテキスト 8%
    「うちのショッピングサイト全体を点検して改善して」
    ↓ 6 つに分けて同時に任せる

    「動作を見る」を押してみてください — 6 人がどう分けて働き、それがなぜ良いのかをお見せします。

    同時に (並列)
    6 人がいっぺんに働きます — 一人で順番にやるより速いです。
    メインはきれい
    それぞれ自分の空間で働き「要約」だけ報告 → メインの会話が散らかりません。
    それぞれ専門
    一人が一つのことに集中 → より正確で安全です。
    • 自分専用の作業空間（コンテキストウィンドウ）を持ちます。お手伝いが長く調べても、メインの会話は散らかりません。
    • できることを制限できます。例: 「コードレビュー」のお手伝いは読むだけで直せないように — より安全。
    • 標準のお手伝いに Explore(探索)・Plan(計画)・General-purpose(汎用) などがあります。それ以外は自分で作れます (.claude/agents/)。

    バー・トークンの数値は理解を助ける例です（実際の値は作業ごとに変わります）。複数のセッションを同時に動かして協力する「エージェントチーム (agent teams)」は、別の実験的機能です。（出典: 公式ドキュメント）

    ボタンを押すと、Claude が画像をどう「読み取る」のかお見せします。

    ログイン

    パスワード

    • ✓構造の把握 — メール・パスワードの入力欄 2 つ
    • ✓色の検知 — 青いボタン「ログイン」

    なぜキャプチャを貼るの?

    言葉より速い
    画面を長々と説明しなくても、そのまま。
    正確な状態
    実際に見えているその姿を直接確認。
    デザインの通り
    デザイン案を渡せば、そのレイアウト・色で。
    エラー画面
    エラーのキャプチャを渡せば原因を早く。

    • とても小さい文字は間違えることがあります。大事な数字・文字はテキストでも一緒に書いておくと安全です。
    • 動く GIF は最初の場面だけ見ます。JPG・PNG などのふつうの画像をおすすめします。

    貼り方: キャプチャを入力欄にドラッグして置くか貼り付け (Cmd/Ctrl+V)、または画像ファイルのパスを書けば OK です。

    自動化 (フック · Hooks)

    フックはひとことで言うと「自動ルール」です — 特定の瞬間に決まった動作が毎回自動で実行されます。

    たとえお店のドアの呼び鈴や、スマホの「23 時に自動でおやすみモード」のように——「○○が起きたら → 自動で△△」を設定。（CLAUDE.md が「お願い」なら、フックは「ルール」）

    「自動返信メール」のように、あることが起きると自動で動きます。

    ご安心を自分で作る必要はありません。 設定方法は少し技術的ですが、非エンジニアは作り方を知らなくても大丈夫 — Claude に 「ファイルを直すたびに自動で整理するフックを作って」のように言葉でお願いすれば、Claude が代わりに作ってくれます。このページは「こういうものがある」とわかれば十分です。

    どう動くか いつ → 何を → どうやって

    1
    イベント (いつ)
    例: ファイルを保存した直後
    2
    フィルター (何を)
    例: それが大切な書類なら
    3
    作業 (どうやって)
    例: 自動でバックアップを1つ作る

    こんなときに便利

    自動バックアップ
    大切なファイルを直すたびに、自動でバックアップを。
    保護
    うっかり大切なファイルを消してしまうのを、あらかじめ防ぐ。
    通知
    作業が終わったら通知を送る。
    • 設定ファイル(settings.json)に書きます。全プロジェクト用・このプロジェクト用・個人用など、いくつかの段階で置けます。
    • 毎回保証されます。CLAUDE.md が「お願い」なら、フックは「ルール」 — 忘れずにいつも実行されます。
    • 数十種類のタイミングに対応します。ファイル変更・ツール実行の前後・セッション開始・圧縮の前後などさまざまな瞬間。/hooks で確認。

    結論CLAUDE.md が『お願い』なら — フックは『ルール』：決めた瞬間に毎回自動で実行されます。

    設定がやや技術的なので、Claude に「ファイルを直すたびに整形するフックを作って」のようにお願いすれば代わりに作ってくれます。（出典: 公式ドキュメント、2026年6月時点）

    プランモード

    すぐ直す前に、Claude が「何をするか」計画を先に見せて、自分の承認を受けます。

    大きな変更や慣れない作業で、「見当違いの方向」を前もって防げます。流れを再生してみてください。

    1
    プランモードに切り替え
    Shift+Tab または /plan — これで読むだけで、直しません
    2
    Claude が計画を提示
    「この 3 つのファイルをこう変えます」 — 影響と順序を前もって見せる
    3
    自分が確認 · 修正
    「footer も抜けてるね」のようなフィードバックでもう一度整える
    4
    承認したら実行
    「OK、実行して」 → そのとき一気にファイルを直します

    結論すぐ直して的外れになる代わりに — 先に計画を見て承認すれば大きな変更も安全です。

    プランモードは「読み取り専用」なので安全です。権限モード（「権限モード」タブ）の一種です。（Windows の一部のターミナルでは Shift+Tab の代わりに /plan をおすすめ）

    権限モード

    Claude がどこまで自分で進めるかを、自分で決めます。

    たとえアシスタントにどこまで任せるか——「何でも先に聞いて」 ↔ 「いつものことは自分で」。最初は確認多め、慣れたら緩める。

    かんたんに言うと · Claude が自分のコンピュータで何か（ファイルを直す・コマンドを実行する）をするとき、「これやっていいですか?」とどのくらい頻繁に聞くかを決める設定です。最初は毎回聞くのが安全で、慣れてきたらよくやる仕事は自動でやらせるように緩められます。車の「マニュアル ↔ オートマ」のようなものです。

    Claude Code では Shift+Tab で基本モード（権限リクエスト・編集の承認・プラン）を循環します。「自動・スキップ」のような自律モードは別のオプションが必要な場合があります。モードを押して「動作を見る」で違いを確認してください。

    ← 安全 · 確認が多い自律 · 確認が少ない →

    結論モードごとに自動で実行する範囲が違います — 最初は『権限リクエスト』で安全に、慣れたらよく使う操作を緩める。

    この 5 つはメニューに見えるモードです。自動化専用に「事前承認したツールだけ許可 (dontAsk)」モードも別にあります。どのモードでもシステム全体の削除 (rm -rf / · rm -rf ~) のような極端な破壊は、もう一度確認を取ります（「サーキットブレーカー」 — 自動では実行されません）。気に入らなければ「巻き戻し」（「巻き戻し」タブ）で復元できます。

    巻き戻し (チェックポイント)

    Claude は、自分が送ったリクエストごとに、その瞬間の状態を自動保存しています。気に入らなければ巻き戻せます。

    「タイムトラベル」のように、以前の時点へ行けます。点を押して巻き戻してみてください。

    状況: 「デザインをモダンに変えて」 → ファイルが 10 個変わったけど気に入りません。どう巻き戻しましょう?

    上のタイムラインの点を押したあと、復元の方法を選んでください。

    結論git を知らなくても — 気に入らなければ前の時点に一度で戻せます。（コードだけ／会話だけ／両方）

    • 開き方/rewind、または入力欄が空のときに ESC を2回 — これまで送ったリクエストの一覧が出ます。（入力欄に文字があると、2回押すと先に文字が消えます。）
    • 3 種類の復元コード+会話を一緒に / 会話だけ(コードは維持) / コードだけ(会話は維持)。
    • 限界rm·mv のようなターミナルコマンドで変わったもの、インストール(npm install)・外部の変更は巻き戻せません。

    チェックポイントは「セッション内の素早い巻き戻し」(約30日保管)、git は「永続的なバージョン管理」 — 役割が違います。長く保管したいなら git にコミットしてください。（出典: 公式ドキュメント、2026年6月時点）

    データ & プライバシー

    「自分のコード・データが AI の学習に使われますか?」 — いちばん多く心配される部分です。

    アカウントの種類によって方針が大きく違います。正確には公式ページで必ず確認してください（下のリンク）。

    個人用 (Free·Pro·Max)

    claude.ai 個人アカウント
    学習への利用ユーザーに選択権があります — 設定で確認・変更可能
    設定の場所claude.ai/settings/data-privacy-controls

    業務用 (Team·Enterprise·API)

    会社・開発者アカウント
    学習への利用基本的に学習には使われない
    API基本的に会話内容を保存しません、学習には未使用（一部の新しいモデルは30日保持が必要）

    • 例外: 安全レビューでフラグされた会話学習をオフにしても、安全・ポリシーのレビュー目的では使用・保管されることがあります。

    コンテキストはなぜ管理が必要？

    コンテキスト（文脈）は Claude の 「机」です。机が散らかると、仕事が 遅くなり、高くつき、ミスが増えます。だからこそ、こまめに片づけることが大切です。

    同じ作業を、放っておくセッションとこまめに片づけるセッションの両方に送ってみましょう。コンテキストがたまるほど、2つの結果がどう分かれるか比べてください。

    ためるだけ 放置
    机（コンテキスト）12%
    作業ごとに片づけ 管理
    机（コンテキスト）12%

    「次の作業を送る」を押して — 同じ頼みを両方のセッションに送り、結果を比べてください。

    片づけ方 — こうやって管理します

    新しい仕事の開始 = /clear
    別の作業に移るとき、机を空にします。（「セッション & フォルダ」タブ）
    長くなったら /compact
    要点だけ要約してスペースを空けます。（「圧縮」タブ）
    1 セッションに 1 つの仕事
    いくつもの仕事を混ぜません — 混ざると混乱します。
    必要なものだけ載せる
    使わないファイルや内容は、わざわざ入れません。
    • なぜ遅く・高くなるの？Claude は答えるたびに机の上すべてを読み直します。積み上がるほど毎回時間がかかり、トークン（＝費用）も増えます。
    • なぜ正確さが下がるの？コンテキストが長すぎると、途中にあった大事な内容を取りこぼしやすくなります。必要なものだけ残せば、より正確になります。
    • 放置すると自動圧縮が来ます。いっぱいになると Claude が自動で要約（自動圧縮）しますが、そのとき詳細の一部が消えることがあります。先に片づければ、自分でコントロールできます。（「コンテキストウィンドウ」「圧縮」タブ）

    ひとことで言えば — きれいな机 = 速くて正確な仕事。 作業が変わるときは /clear、長くなったら /compact だけ覚えておけば大丈夫です。

    圧縮 (compact)

    会話が長くなると「圧縮」が起こります。長く積み上がった内容が、短い要約 1 つに変わります — 机の上の書類を、要点だけ 1 枚にまとめるのと似ています。

    たとえ一日のレシートを「本日の合計」の一行にまとめて引き出しを空けるように——長い会話を短い要約に縮めて場所を空けます。

    下はある作業の会話の記録です。「圧縮する」を押して、内容が実際にどう減るのか見てみましょう。

    圧縮前 — 会話の記録 (長く積み上がる)

    自分「イベント参加者名簿から重複を除いて、日付順に整理して。」
    ファイル参加者名簿.xlsx を読んだ (参加者 800 名)
    途中結果重複チェックの結果 — 同じ人 32 件を発見 … (名前が長々とずらり …)
    Claude「重複 32 件を 1 つにまとめて、申込日順に並べ替えました。これこれの項目が … (長い説明が続く)」
    途中結果整理した表のプレビュー — 768 行 (長い表がずらり …)
    Claude「整理完了。768 名、日付順にそろえたファイルを保存しました。」
    圧縮前 約 12,000トークン→圧縮後 約 1,400トークン 約 88% 節約 例の値
    要約 1 マスに
    ユーザーが「参加者名簿の重複削除＋日付順の並べ替え」を依頼。参加者名簿.xlsx（800 名）を確認 → 重複 32 件を発見 → 1 つにまとめて申込日順に並べ替え。768 名に整理して保存完了。

    残したもの

    核心は保存
    自分の依頼と意図
    処理したファイル
    何をどうやったか
    最終結果

    捨てたもの

    空間の節約
    長い重複名の一覧
    途中段階の冗長な説明
    整理した表のプレビュー

    このように核心だけ残して縮めるので、圧縮の後も「どんな作業をどうやったか」は引き継がれます。ただし捨てられた詳細な一覧（例: 重複した名前の全リスト）がまた必要になれば、Claude がそのファイルをもう一度読みます。トークンの数値は理解を助ける例です（実際の値は状況ごとに変わります）。コンテキストウィンドウで自動的に起こる圧縮は「コンテキストウィンドウ」タブで見られます。

    考える深さ (effort)

    考える深さ（effort）は、Claude が答える前にどれだけ深く考えるかを決める設定です。深く考えるほど難しい問題により正確になりますが、より遅く高くつきます。

    たとえお茶を淹れるよう——さっと一煎（低）vs じっくり濃く（高・最高）。深く考えるほど正確ですが、時間と費用がかかります。

    低 · 中 · 高 · 超高 · 最高の段階があります（デフォルトは「高」）。深く考えるほど「思考（thinking）トークン」を多く使います（「トークン」タブ）。選んで違いを見てみましょう。

    難しい問題の正確さ
    応答の速さ
    トークンのコスト

    選んだぶんだけ「考える過程」が深まります 押すと長さが変わります

    結論簡単な作業は『低』（速くて安い）、難しい推論は『高/最高』（正確だが遅くて高い）。

    簡単な仕事は「低」で（速くて安い）、込み入った分析・推論は「高/最高」で。変え方: /effort または設定（「設定」タブ）。例 のバーは相対的な比較です。（出典: Anthropic 公式 — support.claude.com モデル・effort 設定）

    上級 · ultracode — /effort メニューには ultracode もあります。最高（xhigh）の推論に加えて、難しい作業では Claude が複数のヘルパーを自動的に動員（ワークフロー）して処理します。そのセッション限定のモードで、トークンを多く使うので普段は不要、とても難しい作業のときだけ選びます（Fable 5・Opus 4.8 など）。

    目標達成 (goal)

    目標（goal）は、「完了条件」だけ与えれば、その条件になるまで Claude が自分で繰り返す機能です。毎回もう一度頼む必要がありません。

    たとえ「味が決まるまで味見する」ように——条件を満たすまで自動で繰り返します。（loop は「決まった時刻ごと」）

    一度やって終わりではなく、毎ターンの後に「条件を満たしたか」を自分で判定し、満たしていなければまたやります。下で見てみましょう。

    目標の例「この案内文の誤字が 0 個になるまで直して」
    条件になるまで自動で繰り返す

    「目標スタート」を押してみてください。

    結論毎ターン「もう一回」を押す代わりに — goal は条件を満たすまで自分で繰り返します。

    条件の充足は別の判定で毎ターン確認し、満たされたら自動で止まります（無限ループ防止の仕組みもあります）。変え方: /goal。（出典: Anthropic 公式 — code.claude.com/docs goal）

    繰り返し実行 (loop)

    繰り返し（loop）は、決めた間隔ごとに同じ作業を自動でもう一度実行する機能です。例: 10 分ごとに「新しい注文が入ったか確認して通知」。

    たとえ「毎朝 7 時に水やり」のように——決まった間隔で自動でもう一度。（goal は「条件を満たすまで」）

    目標（goal）が「条件になるまで」なら、繰り返し（loop）は「決めた時間間隔で」です。間隔を選んでスタートしてみましょう。（シミュは時間を早送りして見せます。）

    実行ログ

    間隔を選んで「繰り返しスタート」を押してみてください。

    結論人が10分ごとに自分で確認する作業を — loop は寝ている間も自動で繰り返します。（goal は「できるまで」、loop は「決めた間隔ごと」）

    セッションが開いている間、繰り返されます。変え方: /loop 10m "新しい注文を確認" のように。（出典: Anthropic 公式 — code.claude.com/docs scheduled-tasks）

    ワークフロー (workflows) 上級

    ワークフロー（workflows）は、とても大きな仕事を Claude が自分で「作戦」を立てて、複数のお手伝い（サブエージェント）を同時に動かし、一気に処理する機能です。

    サブエージェントが「一人に任せる」なら、ワークフローは「複数人を同時に調整」します。大きなレビュー・整理・調査に強いです（Max・Team などの上位プラン）。

    例「契約書8 件を項目別に同時にレビューして」 → Claude が作戦を立てて、お手伝いを複数人いっぺんに投入

    「同時作業を見る」を押してみてください。

    非エンジニアにはやや上級の機能です — 「こんなこともできる」とだけ知っておけば十分です。一度に多くのお手伝いを使うのでトークンを多く消費し、大きな仕事にだけ向いています。（出典: Anthropic 公式 — claude.com 動的ワークフロー紹介）

    右の値を押して変えてみてください。

    診断 (doctor)

    診断（doctor）は、何かうまくいかないときに、インストール状態と設定を一度に点検してくれるツールです。「うまくいかないとき、まずやってみること」。

    車の点検ランプのように、どこが問題かを指摘してくれます。診断を実際に走らせてみましょう。

    /doctor 点検中
