Claude Code とは?

コードを自分で書かなくても、いつもの言葉でお願いすればファイルを読んで直して実行してくれる AI ツールです。

ふだん使うチャット（ChatGPT・Claude チャット）と何が違うのか、まずは目で見て比べてみましょう。

チャット vs Claude Code 同じお願い — 押してみてください

Claude チャット アドバイス・コード片

うちのホームページに予約機能を追加して

Claude Code 動く成果物

うちのホームページに予約機能を追加して

大事なのは「答えてくれるか」ではなく、自分のプロジェクトで実際に作って、動くところまでやってくれるかです。最近のチャットも文章やコードを直してくれますが、それは「チャットの中で見せてくれるだけ」 — 複数のファイルを直接作り、つなぎ、実行して「動く成果物」を完成させるのが Claude Code です。（どちらも同じ Claude モデルを使っています。）

Cowork とは？

Cowork は、コードではなく 「オフィス業務」を代わりにこなしてくれる Claude です。目標を伝えるだけで、自分のファイルを直接扱って完成した成果物（Excel・文書など）を作ってくれます。

Claude Code と 同じエンジンですが、ターミナルではなく デスクトップアプリで使う 非エンジニアの業務向けです。下で実際の動きを見てみましょう。

3 つの違い ひと目で

Claude チャット
聞くと 答え・下書きをくれます。自分のファイルは直接さわれません。
例 「レポートはどう書けばいい？」

自分のファイルで業務を代わりに完成させます。非エンジニア向け・デスクトップアプリ。
例 「この領収書を Excel に整理して」
Claude Code
開発作業を代わりにこなします。エンジニア向け・ターミナル。
例 「ログインのバグを直して」

Cowork は Claude デスクトップアプリ（macOS・Windows）の 1 つのモードです（Chat・Code と並んで）。自分が接続したフォルダだけを読み・直し・作ることができ、実行前に 計画を見せて承認を受けます（「確認して進む」推奨）。ファイルを 完全に削除するときは必ずもう一度確認します。Pro・Max・Team・Enterprise プランに含まれます。（出典: Anthropic 公式 — claude.com/product/cowork · 正確な画面・機能は公式アプリに準じます）

どこで使う

Claude Code はいろいろな環境で同じように使えます。CLAUDE.md や設定はどこでも共有されます。

初めてなら「デスクトップアプリ」か「VS Code 拡張」がいちばん簡単です。「ターミナル」(黒い命令ウィンドウ)は慣れた人だけでOK。

自動圧縮 (auto-compact) とは?

会話が長くなってコンテキストウィンドウが上限に近づくと、Claude が自分でこれまでの会話を要約して空間を作ります。上の「自動圧縮を見る」で、いっぱいになっていく途中で要約によって減る過程を見られます。圧縮が起きると、上のマスに緑色で強調された「圧縮で要約された情報」が 1 マスでき、下に減った量を表示します。

いつ始まる? コンテキストウィンドウがほぼ満杯になったとき（上限の近く）に自動で始まります。正確な基準の % は公式には公開されていません（「上限に近づくと」とだけ案内）。圧縮の後もシステム指示・CLAUDE.md・メモリ・MCP ツールは自動で読み込み直されますが、最初に表示されていたスキルの一覧は補充されず、「すでに使ったスキル」だけが残ります。手動の /compact も同じ仕組みです。（出典: 公式ドキュメント、2026年6月時点）

セッション & フォルダ

セッションは一度の連続した会話です。そのセッションのコンテキストウィンドウに、すべてが積み上がります。

コンピュータに詳しくなくても大丈夫です。順番に — まずフォルダを決めて、その中でセッションを始めます。

ステップ1 — まず作業フォルダを決める セッションはフォルダの中で始まります

ボタンを押すと、Claude が「どのファイルを見ることになるか」が表示されます。

トークン

トークンは AI が文章を読む、いちばん小さなかけらです（単語より小さいことも）。タクシーのメーターのように文章が長いほど上がり、料金や利用上限はこれで計算されます。

AI は文字をそのまま読むのではなく、トークン単位に分けて読みます。実際に入力してみてください。

なんでもいいので文を入力 直接打つか、例を押してみてください

同じ意味なのに日本語のトークンのほうがずっと多いでしょう? 同じお願いでも、どの言語で書くかによってトークン（=空間・コスト）が変わります。

日本語 vs 英語 — 同じあいさつなのに

例 トークン数は概念理解のためのおおよその値です（日本語 ≈ 1.5/文字、英語 ≈ 4文字/トークン）。実際の値はモデルのトークナイザーによって少し変わります。

モデルを選ぶ (model)

Claude にはいくつかのモデルがあります。賢さ · 速さ · コストは互いにトレードオフの関係です。

賢いほど、たいてい遅くて高価です。仕事に合わせて選べば大丈夫。下で選んでみてください。

知能 (難しい問題を解く)
速さ (答えが速く出る)
コスト効率 (安い)
こんなときに使う

    Claude Code では /model で変えられます。迷ったらバランスの良い Sonnet で始めましょう。例 バーは相対比較であって、正確な数値ではありません。

    K, M ってどんな単位?

    K = 千
    1K = 1,000 トークン
    例: 200K = 200,000 トークン
    M = 百万
    1M = 1,000,000 トークン
    だいたい本数冊分 — 日常の利用には実質たっぷり

    モデル別のコンテキストウィンドウ (2026年6月時点)

    モデルコンテキストウィンドウひとこと
    Opus 4.8最大 1M (100万)いちばん賢い
    Sonnet 4.6最大 1M (100万)バランス · おすすめの基本
    Haiku 4.5200K (20万)いちばん速くて安い
    • 有料プランで最新の Opus・Sonnet を選ぶと最大 1M コンテキストまで使えます。ただし 1M が使えるかはプランや設定によります（自動ではありません）。Haiku のような基本モデルは 200K です。
    • 日常の業務なら 200K でも十分です。1M は大きなプロジェクトや、とても長い作業で力を発揮します。
    • ウィンドウが大きいほど多く覚えますが、少し遅く、精度が落ちることがあります。必要なものだけ乗せるのが、いちばん良い品質を出します。

    正確な料金・使用量はプランや時期によって変わります — claude.com/pricing で確認してください。

    CLAUDE.md の例

    # 回答のルール - いつも日本語で、ていねいに - コードより説明を先に # このプロジェクト - 紹介ページは pages/about フォルダにあります - 色は青系で統一
    • 200行以下で短く。長すぎると、かえって守られにくくなります。
    • いくつかの場所に置けます。自分のコンピュータ全体 (~/.claude/CLAUDE.md) + プロジェクトごと (./CLAUDE.md) が合わさって一緒に読み込まれます。
    • チームと共有されます。プロジェクトの CLAUDE.md は一緒に使うルール。個人設定は CLAUDE.local.md。

    作成: /init (プロジェクトを分析して自動生成) · 編集: /memory。「Claude が自分で書くメモ」は次の章 (08 メモリ) で。

    メモリ

    自動メモリは「Claude が自分で書くメモ」です。指示しなくても、学んだことを記録しておきます。

    CLAUDE.md が自分の書くメモなら、これは Claude が書くメモです。どう保存され、また使われるのか再生してみましょう。

    1. 自分がひとこと言う
    「これからレポートはいつも表で整理して」
    ↓
    2. Claude がメモファイル(.md)に記録
    ~/.claude/projects/…/memory/ に保存
    ↓
    3. 翌日、新しいセッションを開始
    MEMORY.md を自動で読み込み直す (自分が言わなくても)
    ↓
    4. 自動で表に整理してくれる
    「レポートを表にしました」 — もう一度伝えなくても覚えている

    実際にはこう記録されます

    # MEMORY.md (Claude が自動で作成) (ボタンを押すと、記録されていく様子が表示されます)

    メモリはふつうのマークダウンファイルなので、/memory で直接見て直せます。（自動メモリは Claude Code v2.1.59 以上）

    「複数のセッションで以前の内容を参照できますか?」 — 非エンジニアの方がいちばん多く聞く質問

    • セッションは互いに独立しています。新しいセッションは以前の会話の記録をそのまま持ってきません（きれいに開始）。
    • 代わりにメモリ・CLAUDE.md が「橋」の役割をします。セッションを始めるたびに自動で読み込み直すので、以前学んだ好みやルールは引き継がれます。
    • 特定の古い会話をまるごと続けたいなら/resume で一覧から選んで復活、直前の会話なら claude --continue で続けられます。
    • 自分のコンピュータにだけ保存されます。クラウドや他の端末・同僚と自動共有はされません。チームで分かち合うルールは CLAUDE.md に書いて共有します。

    まとめ: 一つの会話をそのまま続ける = /resume、学んだ点を次の会話へ引き継ぐ = メモリ・CLAUDE.md。（出典: 公式ドキュメント、2026年6月時点）

    スキル (Skills)

    スキルはひとことで言うと「自分のレシピ」です — よく使う作業の手順を書いておき、一度作ればずっと再利用できます。

    毎回ゼロから説明するのと、スキルで一度決めておくのとの違いを再生してみましょう。

    スキルなし 毎回くり返し

    スキルあり 一度だけ定義

    スキルは必要なときだけ開く方式なので、使わないときはコンテキストウィンドウをほとんど占めません。CLAUDE.md が「いつもオンのメモ」なら、スキルは「必要なときに開くマニュアル」です。

    どんな見た目?

    # .claude/skills/週報/SKILL.md --- name: 週報 description: 週次レポートを会社の様式で作成 --- # 手順 1. 今週やったことを集める 2. 会社の様式（見出し・表）に合わせる 3. 要点のまとめを一番上に置く

    呼び出し: /週報 のように直接呼ぶか、関連する作業なら Claude が自動で適用。あらかじめ作られたスキルはプラグインとしてインストールすることもできます（「プラグイン」タブ）。

    MCP (ツール連携)

    MCP は Claude が他のアプリやサービス（Jira・Google ドライブなど）を直接使えるようにつなぐものです — アプリ同士でアカウントを連携するようなイメージです。

    MCP = Model Context Protocol (モデルが外部とつながる標準)。ボタンを押すと連携の過程を見られます。

    かんたんにあるアプリに「Google でログイン」をつなぐと、そのアプリが自分の Google を使えますよね? MCP もそんな「連携」です — Claude に Jira・Figma のようなツールをつないであげると、Claude がそれを直接開いて仕事します。自分でいちいちコピペしなくても済むんです。

    「ひとつずつ連携してみる」を押してみてください。

    こんなものを連携します (例)

    Jira
    課題を読んで作って、状態を変えます。
    例 「このバグを Jira チケットに」
    Figma
    デザイン案を読んでコードに。
    例 「この Figma の通りに作って」
    ブラウザ
    実際の Web ページを開いて確認。
    例 「このページがちゃんと開くか確認」
    その他
    GitHub・Slack・Notion・DB など。
    例 社内ツールと連携

    連携したツールは /mcp で確認。ツールが多いほど、その説明もコンテキストを少しずつ占めるので、必要なものだけ連携してください。MCP を複数まとめて配布するのが「プラグイン」です（「プラグイン」タブ）。

    プラグイン

    プラグインはひとことで言うと「機能のセット箱」です — いろいろな機能を一度にインストールする束（スマホの「アプリ」のように）。

    スキル・ツール連携(MCP)・自動化(フック)などをひとつひとつ別々にそろえる代わりに、誰かがうまく作っておいた束をまるごと受け取って使うわけです。

    かんたんにスマホに「写真補正アプリ」を一つ入れると、フィルターやツールが一気についてきますよね? プラグインもそうです。 これまで見たスキル・MCP・サブエージェント・フックを一つの束としてインストールするんです。（それぞれが何か全部わからなくても大丈夫 — ここでは「束」だということだけわかれば十分です。）

    サブエージェント

    サブエージェントはひとことで言うと「仕事を任せる社員」です — 特定の仕事を任せると自分専用の作業空間で処理し、結果だけを報告します。

    核心はシンプルです — 「仕事を分けて任せる」。

    わかりやすいたとえ · 自分がチームリーダーだと思ってください。大きな仕事を一人で全部やらず、専門のメンバーに「これは君が調べてきて」「これは君が確認して」と分けて任せますよね。メンバーはそれぞれ自分の机で働き、結果だけ報告します。自分の机（メインの会話）はきれいに保たれます。

    実際に見る — 6 人に分けて任せる 押して再生

    メインの会話 (自分と Claude)コンテキスト 8%
    「うちのショッピングサイト全体を点検して改善して」
    ↓ 6 つに分けて同時に任せる

    「動作を見る」を押してみてください — 6 人がどう分けて働き、それがなぜ良いのかをお見せします。

    同時に (並列)
    6 人がいっぺんに働きます — 一人で順番にやるより速いです。
    メインはきれい
    それぞれ自分の空間で働き「要約」だけ報告 → メインの会話が散らかりません。
    それぞれ専門
    一人が一つのことに集中 → より正確で安全です。
    • 自分専用の作業空間（コンテキストウィンドウ）を持ちます。お手伝いが長く調べても、メインの会話は散らかりません。
    • できることを制限できます。例: 「コードレビュー」のお手伝いは読むだけで直せないように — より安全。
    • 標準のお手伝いに Explore(探索)・Plan(計画)・General-purpose(汎用) などがあります。それ以外は自分で作れます (.claude/agents/)。

    複数のセッションを同時に動かして協力する「エージェントチーム (agent teams)」は、別の実験的機能です。（出典: 公式ドキュメント）

    Claude に画像を貼り付け

    Claude は画像を見ることができます。 スクリーンショットを貼れば、言葉で説明するよりずっと速く正確です。

    ボタンを押すと、Claude が画像をどう「読み取る」のかお見せします。

    ログイン
    メール
    パスワード
    ログイン
    パスワードが違います

    Claude が認識したもの

    • ✓文字を読む — タイトル「ログイン」
    • ✓構造の把握 — メール・パスワードの入力欄 2 つ
    • ✓色の検知 — 青いボタン「ログイン」
    • ✓状態の認識 — 赤いエラー「パスワードが違います」
    → Claude: 「ログイン画面で、パスワードのエラー状態ですね。検証ロジックを確認して直してみます。」

    なぜキャプチャを貼るの?

    言葉より速い
    画面を長々と説明しなくても、そのまま。
    正確な状態
    実際に見えているその姿を直接確認。
    デザインの通り
    デザイン案を渡せば、そのレイアウト・色で。
    エラー画面
    エラーのキャプチャを渡せば原因を早く。
    • 画像を「クリック」はできません。画面を分析するだけで、絵の中のボタンを直接押すことはしません。
    • とても小さい文字は間違えることがあります。大事な数字・文字はテキストでも一緒に書いておくと安全です。
    • 動く GIF は最初の場面だけ見ます。JPG・PNG などのふつうの画像をおすすめします。

    貼り方: キャプチャを入力欄にドラッグして置くか貼り付け (Cmd/Ctrl+V)、または画像ファイルのパスを書けば OK です。

    自動化 (フック · Hooks)

    フックはひとことで言うと「自動ルール」です — 特定の瞬間に決まった動作が毎回自動で実行されます。

    「自動返信メール」のように、あることが起きると自動で動きます。

    ご安心を自分で作る必要はありません。 設定方法は少し技術的ですが、非エンジニアは作り方を知らなくても大丈夫 — Claude に 「ファイルを直すたびに自動で整理するフックを作って」のように言葉でお願いすれば、Claude が代わりに作ってくれます。このページは「こういうものがある」とわかれば十分です。

    どう動くか いつ → 何を → どうやって

    1
    イベント (いつ)
    例: ファイルを保存した直後
    2
    フィルター (何を)
    例: それがコードファイルなら
    3
    作業 (どうやって)
    例: 自動でコードの整形・チェックを実行

    こんなときに便利

    自動の整理
    直すたびに自動でコードを整形。
    保護
    危険なコマンド(rm など)をあらかじめ防ぐ。
    通知
    作業が終わったら通知を送る。
    • 設定ファイル(settings.json)に書きます。全プロジェクト用・このプロジェクト用・個人用など、いくつかの段階で置けます。
    • 毎回保証されます。CLAUDE.md が「お願い」なら、フックは「ルール」 — 忘れずにいつも実行されます。
    • 26 種類を超える時点に対応します。ファイル変更・ツール実行の前後・セッション開始など。/hooks で確認。

    設定がやや技術的なので、Claude に「ファイルを直すたびに整形するフックを作って」のようにお願いすれば代わりに作ってくれます。（出典: 公式ドキュメント、2026年6月時点）

    プランモード

    すぐ直す前に、Claude が「何をするか」計画を先に見せて、自分の承認を受けます。

    大きな変更や慣れない作業で、「見当違いの方向」を前もって防げます。流れを再生してみてください。

    1
    プランモードに切り替え
    Shift+Tab または /plan — これで読むだけで、直しません
    2
    Claude が計画を提示
    「この 3 つのファイルをこう変えます」 — 影響と順序を前もって見せる
    3
    自分が確認 · 修正
    「footer も抜けてるね」のようなフィードバックでもう一度整える
    4
    承認したら実行
    「OK、実行して」 → そのとき一気にファイルを直します

    プランモードは「読み取り専用」なので安全です。権限モード（「権限モード」タブ）の一種です。（Windows の一部のターミナルでは Shift+Tab の代わりに /plan をおすすめ）

    権限モード

    Claude がどこまで自分で進めるかを、自分で決めます。

    かんたんに言うと · Claude が自分のコンピュータで何か（ファイルを直す・コマンドを実行する）をするとき、「これやっていいですか?」とどのくらい頻繁に聞くかを決める設定です。最初は毎回聞くのが安全で、慣れてきたらよくやる仕事は自動でやらせるように緩められます。車の「マニュアル ↔ オートマ」のようなものです。

    Claude Code では Shift+Tab でモードを変えます。モードを押して「動作を見る」で違いを確認してください。

    ← 安全 · 確認が多い自律 · 確認が少ない →

    この 5 つはメニューに見えるモードです。自動化専用に「事前承認したツールだけ許可 (dontAsk)」モードも別にあります。どのモードでもシステム全体の削除 (rm -rf / · rm -rf ~) のような極端な破壊は安全装置で防がれます。気に入らなければ「巻き戻し」（「巻き戻し」タブ）で復元できます。

    巻き戻し (チェックポイント)

    Claude は、自分が送ったリクエストごとに、その瞬間の状態を自動保存しています。気に入らなければ巻き戻せます。

    「タイムトラベル」のように、以前の時点へ行けます。点を押して巻き戻してみてください。

    状況: 「デザインをモダンに変えて」 → ファイルが 10 個変わったけど気に入りません。どう巻き戻しましょう?

    上のタイムラインの点を押したあと、復元の方法を選んでください。

    • 開き方ESC を 2 回押すか /rewind — これまで送ったリクエストの一覧が表示されます。
    • 3 種類の復元コード+会話を一緒に / 会話だけ(コードは維持) / コードだけ(会話は維持)。
    • 限界rm·mv のようなターミナルコマンドで変わったもの、インストール(npm install)・外部の変更は巻き戻せません。

    チェックポイントは「セッション内の素早い巻き戻し」(約30日保管)、git は「永続的なバージョン管理」 — 役割が違います。長く保管したいなら git にコミットしてください。（出典: 公式ドキュメント、2026年6月時点）

    サブスク vs API — 何が違う

    サブスク (Pro·Max·Team)

    月額
    料金毎月決まった金額
    上限決まった使用量 — 使い切るとリセットまで待機
    向いているときこつこつ頻繁に使うとき (定額で安心)

    API (従量制)

    使った分だけ課金
    料金使ったトークン分だけ請求
    上限月の上限なし — 使い続けられるが費用は増える
    向いているときたまに使うか、自動化・連携に

    2 つの上限が同時にあります

    • 5時間の上限「直近5時間」に使った量を見ます。時間が過ぎると自動で再び満ちます(ローリング方式)。
    • 週間(7日)の上限「直近7日」基準の、より大きい上限。5時間の上限さえ避ければ無限に使える、というわけではありません。
    • どちらか一方でも満杯になると止まります。5時間・週間の 2 つの上限が一緒にかかっているので、先に満杯になるほうで止まります。その上限がリセットされれば再び使えます。
    • 上限にかかっても作業は消えません。やっていた内容が消えるのではなく「一時停止」です — リセットされたら続きをやれば OK。（急ぐなら上位プランへアップグレード、または追加の使用量を購入）
    • 一か所でまとめて差し引かれます。claude.ai・Claude Code・デスクトップアプリが同じ使用量を分け合います。

    圧縮 (compact)

    会話が長くなると「圧縮」が起こります。長く積み上がった内容が、短い要約 1 つに変わります — 机の上の書類を、要点だけ 1 枚にまとめるのと似ています。

    下はある作業の会話の記録です。「圧縮する」を押して、内容が実際にどう減るのか見てみましょう。

    圧縮前 — 会話の記録 (長く積み上がる)

    自分「イベント参加者名簿から重複を除いて、日付順に整理して。」
    ファイル参加者名簿.xlsx を読んだ (参加者 800 名)
    途中結果重複チェックの結果 — 同じ人 32 件を発見 … (名前が長々とずらり …)
    Claude「重複 32 件を 1 つにまとめて、申込日順に並べ替えました。これこれの項目が … (長い説明が続く)」
    途中結果整理した表のプレビュー — 768 行 (長い表がずらり …)
    Claude「整理完了。768 名、日付順にそろえたファイルを保存しました。」
    圧縮前 約 12,000トークン→圧縮後 約 1,400トークン 約 88% 節約 例の値
    要約 1 マスに
    ユーザーが「参加者名簿の重複削除＋日付順の並べ替え」を依頼。参加者名簿.xlsx（800 名）を確認 → 重複 32 件を発見 → 1 つにまとめて申込日順に並べ替え。768 名に整理して保存完了。

    残したもの

    核心は保存
    自分の依頼と意図
    処理したファイル
    何をどうやったか
    最終結果

    捨てたもの

    空間の節約
    長い重複名の一覧
    途中段階の冗長な説明
    整理した表のプレビュー

    このように核心だけ残して縮めるので、圧縮の後も「どんな作業をどうやったか」は引き継がれます。ただし捨てられた詳細な一覧（例: 重複した名前の全リスト）がまた必要になれば、Claude がそのファイルをもう一度読みます。トークンの数値は理解を助ける例です（実際の値は状況ごとに変わります）。コンテキストウィンドウで自動的に起こる圧縮は「コンテキストウィンドウ」タブで見られます。

    プログラム同士がやり取りする「つなぎ目」。ここでは使った分だけ払う「従量制」の使い方を指します。（「料金プラン」タブ）
    サーバー
    遠くで仕事を処理してくれるコンピュータ。MCP「サーバー」は外部ツールを Claude につなぐ小さなプログラムです。
    .md (マークダウン)
    簡単なルールで飾る「メモ用の文章ファイル」の形式。CLAUDE.md、SKILL.md の .md がこれです。
    JSON
    コンピュータが読む「設定を書く形式」。フックのような設定をここに書きます。(自分で書かなくても Claude が代わりにやってくれます)
    git
    ファイルの変更履歴を記録・巻き戻す「バージョン管理」ツール。巻き戻しの永続保管版。（「巻き戻し」タブ）
    従量制
    月額ではなく「使った分」だけ払う方式 (API)。（「料金プラン」・「使用量の上限」タブ）
    ローリング上限
    「直近5時間・直近7日」のように流れながら再び満ちる使用量の上限。（「使用量の上限」タブ）
    プロトコル
    違うプログラム同士が話すための「共通のルール」。MCP の P がこれです。
